# Forest Agent Dashboard

## Product register

Product.

## Platform

Desktop Obsidian community plugin. The dashboard runs inside a local vault and uses the Obsidian desktop runtime for secure storage and optional network integrations.

## Audience

Privacy-conscious Obsidian power users who want one calm place to move between notes, tasks, inbox summaries, and engineering signals without sending their vault to a hosted dashboard.

## Purpose

Turn a personal vault into a local-first operational cockpit: search what was written, return to frequently used notes, review tasks, and optionally glance at read-only mail and public engineering feeds.

## Positioning

Forest Agent Dashboard combines:

- full-text local note search;
- frequent-note history stored in plugin data;
- local tasks;
- read-only Gmail and QQ Mail metadata views;
- opt-in GitHub and Hacker News signals.

Vault content stays local. External integrations are explicit, optional, and documented.

## Brand personality

Quiet, forested, practical, curious, and personal without being childish. The public identity is **Totoro**, while the visual system uses an original forest-and-letter-T mark rather than any Studio Ghibli character or artwork.

## Product principles

1. Local first: core note and task features work without an account.
2. Read-only by default: mail integrations fetch headers and metadata, never mutate messages.
3. No hidden telemetry: the plugin sends no analytics.
4. Secrets are not settings: credentials use Obsidian secure storage when available.
5. Demo data is synthetic: documentation never contains a real vault, mailbox, token, or address.
6. Calm density: useful information should feel compact, not cramped.

## Anti-references

- No copied Totoro, Studio Ghibli, or other entertainment character imagery.
- No generic purple-gradient AI dashboard styling.
- No decorative animation that competes with reading.
- No screenshots containing personal notes or email.
- No credentials, tokens, app passwords, or OAuth responses in repository files.

## Accessibility baseline

- Visible keyboard focus.
- Semantic buttons and labels.
- High-contrast light and dark themes.
- Reduced-motion support.
- Layout remains usable at narrow Obsidian pane widths.
